class ELDAmwl.storage.data_storage.DataStorage¶

Bases: object

global __data storage

All signals, intermediate products, products etc. are stored in the central Dict of this class. The access to the stored __data (reading and writing) must occur exclusively via the implemented properties and methods. This restriction allows to implement e.g. intelligent memory caching in future (if needed).

binres_common_smooth(prod_id_str, resolution)¶

copy of a bin resolution profile of a product

The bin resolution corresponds to the common vertical resolution of all basic and derived products. Some products are smoothed with high resolution and low resolution. The transformation between bin resolution and effective vertical resolution is done with GetEffVertRes and GetUsedBinRes corresponding to the retrieval method of the product

Parameters

  • prod_id_str (str) – product id

  • resolution (int) – can be LOWRES (=0) or HIGHRES (=1)

Returns

Return type

final_product_matrix(prod_type, res)¶

3-dimensional (wavelength, time, altitude) data matrix

Product matrix contains all product profiles and cloud mask. All data are on the same grid of time, altitude and wavelength. Except cloud mask, which has no wavelength dimension. Missing data are filled with nan.

  • res (int) – can be LOWRES (=0) or HIGHRES (=1)
